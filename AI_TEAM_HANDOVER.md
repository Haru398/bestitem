# 👋 환영합니다! 안티그래비티 AI 팀을 위한 친절한 업무 인수인계서

새로운 컴퓨터 환경에서 업무를 이어받게 된 10인의 AI 팀원 여러분, 진심으로 환영합니다! 
이 문서는 여러분이 길을 잃지 않고 **"우리가 그동안 어떻게 일을 해왔는지"** 아주 쉽고 천천히, 그리고 자세하게 알려주기 위해 작성되었습니다. 마치 아기에게 걸음마를 가르쳐 주듯 하나하나 설명해 둘 테니, 업무를 시작하기 전 반드시 팀원 전체가 이 내용을 완벽하게 숙지해 주시길 바랍니다.

---

## 1. 🏢 우리가 담당하는 프로젝트는 무엇인가요?
우리는 대표님(사용자)을 도와 리뷰 전문 웹사이트(`item.monster`)를 운영하고 있습니다. 
우리의 핵심 임무는 사람들이 제품을 사고 싶게 만드는 매력적인 리뷰 글을 작성하고, 쿠팡 파트너스를 통해 수익을 창출하며, 사이트가 오류 없이 잘 돌아가게 유지보수하는 것입니다.

---

## 2. 📝 글을 작성할 때 "무조건" 지켜야 할 4가지 대원칙

1. **분량은 길고 풍부하게**: 
   - 공백을 제외하고 **최소 2,000자에서 2,500자 이상**의 정성스러운 글을 써야 합니다.
2. **사진 탐색 및 5장 이상 배치 (매우 중요)**: 
   - 대표님께서 사진을 직접 제공해 주시면 그 원본 고화질 사진을 사용하십시오.
   - 만약 대표님께서 **사진을 주지 않고 제품명만 던져주며 포스팅하라고 지시하실 경우**, 여러분이 직접 웹 검색 등을 통해 **해당 제품과 정확히 일치하는 사진을 찾아 5장 이상 확보**해야 합니다.
   - 🚨 *[경고]* 해당 상품과 다르게 생겼거나 아예 연관성이 없는 엉뚱한 사진을 사용하면 절대 안 됩니다.
   - 🚨 *[경고]* 사진이 억지로 늘어나거나 찌그러지면 안 됩니다. CSS에 반드시 `object-fit: contain; max-width: 100%; height: auto;`를 적용하여 원본 비율을 유지하세요.
3. **항상 쿠팡 파트너스 규칙과 저작권을 따라야 합니다**:
   - 남의 블로그나 사이트 글을 그대로 복사해서 붙여넣기(크롤링)하면 절대 안 됩니다. 항상 새로운 창작글을 쓰세요.
   - 글을 작성할 때 반드시 **쿠팡 파트너스 규칙과 저작권을 철저히 준수**해야 합니다.
4. **쿠팡 파트너스 필수 문구는 "게시글 상단"에 배치**:
   - **[법적 의무]** 모든 게시글의 **맨 위(최상단)**에는 무조건 다음 문구를 기재해야 합니다.
     👉 *"이 포스팅은 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다."*
   - 쿠팡 구매 링크와 위젯(iframe)도 빼놓지 말고 적절한 위치에 꼭 삽입하세요.

## 3. ⚙️ 아주 중요한 "3단계 업로드 방식" (우리가 일해온 방식)
우리는 글을 썼다고 해서 **절대로 곧바로 정식 사이트(메인 DB)에 올리지 않습니다.** 대표님께서 먼저 보시고 허락(승인)을 해주셔야만 올릴 수 있습니다. 이 순서를 꼭 기억하세요!

### 👶 1단계: 로컬에서 초안 만들고 '임시 사이트' 빌드하기
- 먼저 `src/app/preview-test/page.tsx` 파일에 우리가 쓴 글을 넣어 화면이 어떻게 나오는지 코드로 만듭니다.
- 그 다음 Next.js를 정적 파일로 변환(`npm run build`)해야 하는데, 이때 주의할 점이 있습니다!
  - *해결법:* 조회수를 세기 위해 만든 `api` 폴더(`/api/stats` 등) 때문에 빌드 에러가 납니다. 빌드하기 직전에 임시로 API 폴더 이름을 바꾼 뒤 빌드해야 합니다.
  - **터미널 명령어 순서**:
    1. `Move-Item -Path "src\app\api" -Destination "src\app\api_temp" -Force` (API 폴더 잠깐 숨기기)
    2. `npm run build` (정적 out 폴더 생성)

### 👶 2단계: Surge 임시 사이트에 올리고 대표님께 검사 맡기
- `out` 폴더가 만들어졌으면, 아래 명령어를 쳐서 **Surge 임시 서버**에 올립니다.
  - `npx surge out bestitem-lab.surge.sh`
- 배포가 끝나면, 숨겨둔 API 폴더를 다시 원래대로 돌려놓습니다.
  - `Move-Item -Path "src\app\api_temp" -Destination "src\app\api" -Force`
- 그리고 대표님께 채팅으로 이렇게 보고합니다. 
  *"대표님! 저희가 `https://bestitem-lab.surge.sh/preview-test` 링크에 임시로 글을 올려두었습니다. 승인해 주시면 정식 사이트에 올리겠습니다!"*

### 👶 3단계: 대표님 승인 후 정식 사이트에 올리기 (진짜 업로드)
- 대표님이 "승인한다"라고 하시면, 우리의 데이터베이스(Supabase)의 `posts` 테이블에 글 데이터를 집어넣습니다.
- 코드 변경 사항이 있다면 GitHub에 `commit` 하고 `push`를 하면, Vercel이 자동으로 정식 메인 사이트(`item.monster`)를 업데이트해 줍니다.
