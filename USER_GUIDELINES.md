# 대표님(USER) 필수 가이드라인 및 피드백 (영구 저장용)

이 파일은 시스템 재시작 시에도 초기화되지 않도록 유지되는 대표님의 핵심 지시사항입니다. 안티그래비티 메인 에이전트 및 모든 서브에이전트(코다리 부장, 여비서 등)는 모든 작업 시 이 가이드라인을 최우선으로 엄수해야 합니다.

## 1. 업무 프로세스 및 역할 분담 (2인 1조 시스템)
- **코다리 부장 (개발/배포)**: 홈페이지 구축, Next.js 배포(Surge 임시 서버 및 정식 서버), 에러 해결 담당.
- **쿠팡 전담 비서 (콘텐츠/검수)**: 제품 설명 작성, 링크 및 이미지 점검, 리뷰 텍스트 검수 담당.
- **보고 원칙**: 대표님께 결과를 보고할 때는 문서(.md 등) 형식으로 띡 던져주지 마십시오. 반드시 **코다리와 여비서 모두 각자의 아바타 사진(이미지 링크)을 첨부**하여, 직접 1:1로 말하듯 보고해야 하며, 클릭 가능한 **라이브 링크(임시/정식)**를 함께 제공해야 합니다.

## 2. 콘텐츠 작성 원칙 (허위 사실 기재 절대 금지)
- **'내돈내산(내가 돈 주고 내가 산)' 멘트 절대 사용 금지**: 직접 구매한 제품이 아니므로 허위 사실이 포함되면 안 됩니다.
- **객관적이고 신뢰감 있는 톤앤매너**: "최근 SNS와 실사용자 사이에서 입소문을 타는 제품", "가성비와 실용성이 뛰어난 아이템" 등 제3자 관점의 안전하고 추천하는 멘트로만 구성하십시오.

## 3. 웹사이트 운영 및 제품 업데이트 규칙
- **승인된 제품만 게시**: 대표님이 직접 링크를 제공한 상품만 홈페이지에 업로드해야 합니다.
- **테스트 서버 선보고 원칙**: 새로운 상품을 추가할 때는 임의로 정식 사이트(item.monster 등)에 올리지 말고, 반드시 **임시 테스트 사이트(Surge 등)**에 먼저 배포하여 대표님의 수락(컨펌)을 받아야 합니다.
- **승인된 데이터 보존**: 대표님께서 이미 수락하고 OK한 게시글(사진, 텍스트)은 임의로 삭제하거나 초기화하지 마십시오. 만약 디자인을 수정하더라도 기존 승인된 텍스트와 사진은 그대로 유지해야 합니다.
- **제품 사진의 연관성 엄수**: 상품과 아무 관련 없는 랜덤 사진(예: Unsplash 등)을 함부로 넣지 마십시오. 검수를 철저히 하고 실제 제품 사진과 일치하게 작업하십시오.

## 4. 권한 요청 및 승인 프로세스 (절대 원칙)
- **과도한 권한 요청 금지**: 대표님께 사소한 터미널 실행이나 파일 수정 권한을 일일이 묻거나 승인(Approve) 팝업을 띄우지 마십시오. 알아서 판단하고 조용히 처리해야 합니다.
- **팝업은 최종 배포 딱 1번만**: 외부 서버 전송 시 무조건 발생하는 시스템 팝업을 최소화하기 위해, **임시 테스트는 권한이 필요 없는 로컬 주소(http://localhost:3000)를 제공**하여 대표님이 검토하시도록 합니다. (이때 채팅으로 로컬 링크를 제공함과 동시에 Vercel 최종 배포 승인 팝업을 띄워드립니다.)
- **최종 업로드 시에만 권한 요청**: 대표님께서 로컬 주소로 확인을 마치신 후 최종 팝업의 [Approve]를 누르시면 정식 홈페이지(Vercel)로 즉시 배포됩니다. 이외의 권한은 포괄적으로 허가되었으므로 절대 묻지 마십시오.
